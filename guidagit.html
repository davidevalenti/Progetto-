GUIDA ALL'UTILIZZO DI GIT

  • Inizializzazione

    Dopo aver avviato Git, occorre specificare il proprio nome e la propria email attraverso i comandi:

    git config --global user.name "nome_utente"
    git config --global user.email "email_utente"
    In seguito occorre creare una cartella vuota dove inserire il "tracker" di Git:
    mkdir nome_cartella
    A questo punto, entrando nella cartella appena creata attraverso il comando
    cd percorso_cartella
    e scrivendo
    git init
    Si è pronti a lavorare.

  • Comandi Utili

    • ls : visualizzo gli elementi visibili nella cartella selezionata
    • ls -a : visualizzo gli elementi visibili e "invisibili" nella cartella selezionata
    • cat file: mostra il contenuto del file
    • touch file: crea un file col nome file nella cartella selezionata
    • programma file : apre un file col nome file attraverso il programma chiamato programma
    • git add file : comando da utilizzare prima del commit per rendere possibile il commit del file
    • git commit -m "Testomessaggio" : esegue un commit descritto all'interno delle virgolette
    • git log : restituisce il log dei commit
    • git status : comando necessario per controllare se sono state effettuate delle modifiche.

  • Integrazione con SourceTree e VSCode

    La directory di cui vogliamo tenere traccia può essere anche impiegata da SourceTree per un approccio prevalentemente grafico. Infatti occorre solo trascinare la nostra cartella nell'apposito spazio (nella barra a sinistra) per inserire un "tracker" delle nostre modifiche. Su SourceTree è inoltre possibile creare nuovi branch (rami) del nostro file, pensabili come i rami che provengono dal tronco di un albero; ma è anche possibile effettuare il Merge/Rebase dei file, ovvero combinare 2 o più file provenienti da branch differenti in un unico documento contenente le modifiche di tutti. Alla stessa maniera, Visual Studio Code si integra perfettamente con SourceTree, permettendo di switchare in tempo reale da un commit all'altro attraverso Click Dx>Checkout> (sulla versione del file che ri-vogliamo)>Okay.
